Load and follow the `no-stub` skill, then sweep your own work this session.

1. List the code files you created or edited this session.
2. For each, scan for stub markers: `TODO`/`FIXME`/`XXX`/`HACK`, `NotImplementedError` / `todo!()` / `throw new Error("not implemented")`, empty `...` or `pass # TODO` bodies, and phrases like "your code here" / "implementation goes here" / "rest of … here".
3. For each hit, **finish it**: replace the placeholder with a real, working implementation (read the types and callers — the answer is usually in the repo). If a path can't be built yet, make it fail correctly with a real error rather than a fake return.
4. If a stub is genuinely intentional (interface / abstract method / agreed-deferred path), keep it and record one line: `NO-STUB: INTENTIONAL — <reason>`.
5. Report what you finished and anything you deliberately left as an intentional stub.

Don't say "done" while a placeholder is still standing in for logic.
